# たばコン

公開URL: https://chameleonjp.codeberg.page/tabakon/

game_slug: `tabakon`

## 概要

「たばコン」は、砂を削り、左バーで画面を上下に見ながら🚬を🦊へ届けるスマホ向けブラウザパズルゲームです。Sand Balls風の「掘って流れを作る」遊びを参考にしつつ、車両配送の構図は使わず、🚬を🦊の回収ゾーンへ届けるゲームとして実装しています。ステージは10種類からゲーム開始ごとにランダムで選ばれます。ステージ1「三つ道」を基準に、ステージ2〜10は左安全路、中央ゲート、右高得点路、二段バー、広い受け皿、細い通路、危険サンド、回転障害物、総合として、砂・壁・掘り抜き導線・危険ゾーン・ゴール前受け皿・バー/ゲート/ローターの配置を明確に差別化しています。

## 操作方法

- 名前を入力して「ゲーム開始」を押します。
- 3、2、1のカウント後にゲームが始まります。
- Canvas上を指でなぞると砂を削れます。
- Canvas左端の縦バーをタップ/ドラッグすると、2画面分あるゲーム世界の上層・中層・下層を見られます。
- 左端24pxのバー領域では砂を削らず、カメラ操作だけを行います。
- 60秒以内に20個以上の🚬を🦊へ届けると成功です。
- ルール画面では色別🚬の点数（白っぽい🚬1000点、赤い🚬1500点、青い🚬1300点、黒い🚬1700点）と特徴を確認できます。
- 危険ゾーンに入った🚬は失われます。黒い🚬は危険ゾーンに近いだけでも失いやすいので注意します。危険ゾーンはCanvas内で赤黒または黒紫の塗り、赤枠、点滅、ラベル「危険」または「×」で表示します。
- リタイアボタンで途中終了できます。

## ランキング仕様

- スコア名: 配達スコア
- 単位: 点
- 順序: 高い方がよい `desc`
- 種別: ベストスコア型 `best`
- ステージ別ランキングは作らず、全10ステージ共通で `tabakon` の配達スコアランキングに送信
- 小数: なし
- 送信: クリア、失敗、リタイア、マイナススコアを含め、ゲーム終了時に自動で1回だけ送信
- Supabase URL: 共通設定 `https://mlpnjgezrnhdxsxolyzj.supabase.co`
- 送信: `submit_score` RPC（`/rest/v1/rpc/submit_score`）
- 取得: `get_best_score_ranking` RPC（`display_name` / `rank_no` 対応）
- 禁止: `/rest/v1/game_scores` への直接insert、旧ランキングテーブル、管理者用キー、サーバー用キー

## 注意事項

- ゲーム本体は `index.html` 1ファイルにHTML/CSS/JavaScriptをまとめています。
- npm、Vite、ビルド環境は不要です。
- 名前は `localStorage` の `tabakon_player_name` に保存しますが、公式ランキングのスコア本体は保存しません。
- SupabaseにはPublishable keyのみを記載しています。
- Canvas表示は320×480のまま、内部世界は320×960です。砂グリッドは48×144で、左バーのゲーム内カメラで表示範囲を切り替えます。48×144グリッドは固定セルサイズではなく、世界サイズから算出した `CELL_W` / `CELL_H` で描画と判定を揃えます。
- ゲーム中と結果画面に選ばれたステージ名を表示します。ランキングはステージ別に分けず、全ステージ共通です。
- 結果画面には届けた点、残り時間、失ったペナルティ、リタイアペナルティ、合計のスコア内訳を表示します。
- RESULT後は `stopLoop()` で描画ループを停止します。
- 入力はタッチ開始位置で固定されます。左端24pxはスクロールバー操作専用、通常エリアは `screenToWorld()` で世界座標へ変換して砂を削ります。ページ自体の縦スクロールではなく、ゲーム内の `cameraY` で上下2画面を切り替えます。10ステージは48×144の縦長用に再設計し、ランキングは従来通り共通ランキングのままです。

## 最新のプレイ改善

- 最下部まで落ちた🚬は、ゴール横の猶予範囲にいない場合、失った扱いで消えます。
- 動かせる🚬が残っていない場合は、制限時間を待たずに結果へ進みます。
- ゴール直上に長い水平固定壁、長い水平バー、横向きゲートを置かない方針です。
- 砂削り後に段差を軽く丸め、階段状の詰まりを減らしています。
- 左スクロールバー上下に▲/▼を表示し、つまみとマーカーを見やすくしています。
- 結果画面の「もう一度」はHOMEへ戻らず、同じ名前でREADYから再開します。
- ステージ5・10の下段バーをゴール直上から少し上へ移し、`validateAllStages()` の警告が出ないようにしています。
- 🚬に淡い影を付け、空色や砂の背景でも各色の🚬を見失いにくくしています。
- リタイアは確認ダイアログを挟み、誤タップでの-5000点を防ぎます。
- 残り時間は10秒以下で小数1位＋赤字になり、終盤の緊張感を出します。
- ランキング通信は8秒で打ち切り、回線不通でも「送信中…」のまま固まりません。
